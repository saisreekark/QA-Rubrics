Structure:

verified against — it replaces the manual ~30-row gold curation that was
blocking the evaluator.

### Strategy shift: label-match evaluation

The evaluator moves from *LLM-judge-of-correctness* to **label-match** —
compare the bot's predicted L1/L2/L3 against the human canonical label.
Implemented this session (`evaluator/runner.py --match-labels`); see
`evaluator/README.md`.

**Feasibility, verified against live data:**
- The *live* I/O sheet bot output is essentially **disjoint** from the
  dump labels (TTR 0, Reopen 0 overlap) — wrong source.
- The dump's own `RCA_Analysis_Output` tab is the right source. Bot⋂label
  overlap: **TTR 4389, Reopen 2595, Escalation 1441, DSAT 324, Quality
  5829**.
- Caveat: `Quality_RCA_Output` / `Workflow_RCA_Output` JSON is populated
  for only ~200 cases in that tab, so the four driver frameworks
  (TTR/Reopen/Escalation/DSAT) are the rich starting point; Quality &
  Workflow scored-match is a follow-on.

## 3. Baseline run + regen path (same-day, 2026-05-29 PM)

**Label-match baseline ran.** All four driver frameworks, ~200 each, one
CSV (`evaluator/runs/2026-05-29T05-21-56Z.csv`):

- **Case-level accuracy: 12.93% (95/735).** TTR 16.0% (32/200), DSAT 13.2%
  (22/167), Reopen 13.1% (26/198), Escalation 8.6% (17/198).
- Error budget is **systemic**, not one framework. **88% of misses are
  full L1 mismatches** (coarse classification, not L3 wording); **~16% of
  misses cite the un-encoded May-14 drivers** the bot can't emit yet.
- This 12.93% is a **strict full-primary-driver match** and is *not*
  comparable to the prior "~50%". It's the anchor for future deltas.
- **Caveat:** the matcher is an unvalidated LLM judge (no match-mode
  judge↔human calibration yet) — directional until calibrated (Phase 2.6).

**Regen path verified** (how to test a prompt/driver change). `--match-labels`
scores the *frozen* `RCA_Analysis_Output` predictions, so prompt edits
don't change what it measures — predictions must be regenerated. The
inputs already exist: the dump's **`Raw Dump` tab** (13,988 rows) is the
exact population that produced those predictions (overlap matches the
bot⋂label counts; transcript populated; all `COL_MAPPING` cols present).
Plan: sample `Raw Dump` → run pipeline → new tab `RCA_regen_test` →
repoint evaluator. Gotchas: **never push regen config to the prod
notebook** (daily scheduler); fix `Case_History ` (underscore) →
`Case History ` (space). See `docs/execution-plan.md` 2.7, memory
`project_regen_path.md`.
